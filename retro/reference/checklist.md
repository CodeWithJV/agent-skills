# Retro Checklist

Use this checklist when running a retrospective.

## Review surface evidence

- What was the user trying to achieve?
- What sequence of actions happened?
- Where did time or tokens get burned?
- What retries or reversals happened?
- What errors appeared?

## Separate symptoms from causes

Symptoms:
- repeated searching
- wrong files opened
- stuck on setup
- confusing tool output

Possible causes:
- weak request framing
- poor skill trigger description
- missing repo orientation
- no deterministic helper script
- missing workflow checkpoint

## Preserve strengths

Before proposing changes, identify:
- what worked well
- what sped things up
- what produced a clean result

Do not damage those strengths with broad changes.

## Suggest changes at the right level

Possible intervention levels:

- user prompting / request framing
- skill improvement
- new skill creation
- workflow change
- checklist
- script / CLI
- repo structure or context hub change

Choose the smallest change that fixes the root cause well.
